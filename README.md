## Тестирование API

В проекте реализованы автоматические тесты с использованием `pytest` и `pytest-django`.

### Используемые инструменты

- `pytest`
- `pytest-django`
- `pytest-cov`
- `rest_framework.test.APIClient`

### Что покрыто тестами

Проверены основные сценарии работы API:

- доступность endpoint `/api/health/`
- доступ активного сотрудника к `/api/network-nodes/`
- запрет доступа для пользователя без статуса сотрудника
- запрет получения токена для неактивного сотрудника
- фильтрация по стране
- поиск по названию
- сортировка по имени
- запрет изменения поля `debt_to_supplier` через API
